Technical details

Ttb

This plugin uses an improved version of ttb (Trace Tool Builder, http://www.erlang.org/doc/man/ttb.html ) for tracing purposes. "Improved" means that this version is currently not included in OTP and comes as part of this plugin. This version is called here "ttbe" ("e" stands for "extended").
Ttb is built on top of dbg and provides some improvements which makes tracing easier. This plugin provides graphical user interface to ttb which makes using it more user friendly.

Tracing

Ttb is started on separate erlang node called "tracing" ( -sname tracing ). It allows tracing tool to be independent from node you want to trace. "Tracing" node is automatically started by erlide when any tracing action is performed for the first time (tracing another node or loading trace data from file).
Traced node is treated as "diskless node", i.e. all data from this node is directly sent to the "tracing" node. After finishing tracing all data is writen to the directory named ttb_upload_erlide_tracing-Timestamp ( Timestamp is in a form of yyyymmdd-hhmmss) into file called nodeName@hostName .
When tracing is finished application reads data from this file using special data handler provided to ttbe:format/2 function which, instead of formatting data, sends it directly to trace plugin which interprets it.
Same action is performed when reading trace results from file.
