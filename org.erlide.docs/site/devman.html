1. Setting up the environment

1.1. Installing Eclipse and Erlide

  • Install Erlang R9 or later, if it isn't already present on your system.

  • Install Eclipse. The recommended Eclipse version is 3.1.1.

  • Point the Eclipse update manager to the Erlide update site at http://erlide.sourceforge.net/update and retrieve the latest version. If your network uses a proxy, fill in the appropriate data in Window->Preferences->Install/Update->Proxy settings.

  • Restart. A dialog will pop up, asking for the path to your Erlang instalation. Enter the path to the top level directory (i.e. $ERL_TOP). Do not restart now, but go to Window->Preferences->Run/debug->String substitution and add a new item, named ERL_ROOT and with the same value as above. Now restart.

  • Done!

1.2. Retrieve code from CVS

Configure Eclipse CVS to use extssh connection method when "ext" is chosen.

In the CVS perspective, connect to the erlide CVS, at erlide.cvs.sourceforge.net:/cvsroot/erlide. Use "ext"-type of connection.

Check out the following modules:

  • erlide-basic-ui

  • erlide-core

  • erlide-jinterface

  • erlide-launching

  • erlide-ui

  • erlide-feature

  • erlide-update

  • erlide-docs

  • erlide-help

  • erlide-erlc

The code for Eclipse 3.2 is in a branch called "convert_32".

1.3. Build

  • After that, everything should compile and you can try starting the newly build Erlide. For this you have to create a run configuration. Choose Run->Run... and Eclipse Application->New. Use a nice, friendly name like "Erlide" and in the plug-ins tab select "Choose from list".

  • Check all the erlide plugins under "workspace plugins" and then "Add required plugins" for a minimal setting. Press validate and add any other plugins that may be needed.

  • Go to the Common tab and check the Show in favorites for both run and debug.

  • Check all the erlide plugins under "workspace plugins" and then "Add required plugins" for a minimal setting. Press validate and add any other plugins that may be needed.

2. Description of plugins

2.1. erlide-feature

The erlide feature, pointing to the plugins.

For alpha releases, the feature's version number is to be forced into the plugins.

Here we can also build an archve with all the plugins.

2.2. erlide-jinterface

A wrapper for the jinterface library. Provides also some extensions to it (like support for Pids).

2.3. erlide-launching

This plugin allows starting Erlang nodes and provides the Backend abstraction.

2.4. erlide-core

Here we define the rest of the non-ui functionality. Most important are:

  • the Erlang model

  • the builder

2.5. erlide-basic-ui

This plugin provides the means to configure the backend (the dialogs for entering the path to the used Erlang instalation).

2.6. erlide-ui

This plugin contains all UI related functionality.

2.7. erlide-update

This is a project for building the update site.

2.8. erlide-docs

This is just a gathering place for all documentation, both internal and external (the latter includes the project's web site).
